门诊预约与智能分诊辅助系统 - 前端静态HTML说明（提交版）

生成日期：2026-04-12

1. 文档说明

本文档用于说明项目中静态 HTML 原型的结构、演示方式，以及与主系统的关系。

2. 原型目录

  • static-html/login.html：登录页原型
  • static-html/index.html：首页概览原型
  • static-html/appointments.html：挂号预约页原型
  • static-html/triage.html：AI 分诊页原型
  • static-html/assets/style.css：统一样式
  • static-html/assets/app.js：前端演示交互脚本

3. 打开方式

进入 static-html 目录，双击 login.html 或 index.html 即可在浏览器查看。

4. 页面说明

  • 登录页：用于演示账号输入、按钮交互与跳转
  • 首页：用于演示公告、统计卡片与导航布局
  • 预约页：用于演示预约表单、时间选择和状态展示
  • 分诊页：用于演示症状输入和建议结果显示

5. 交互机制

页面内交互由 app.js 提供，属于前端本地模拟，不依赖后端 API。

因此表单提交、分诊结果等行为仅用于视觉和流程演示。

6. 与主系统关系

静态原型用于课程作业展示和设计说明；真实业务数据处理以 Vue + Express + MySQL 主系统为准。

7. 演示建议

  • 先从 login.html 开始，演示完整跳转链路
  • 再展示 appointments.html 和 triage.html 两个核心场景
  • 最后说明静态原型与动态系统的差异边界

8. 交付建议

提交时可将 static-html 目录与本说明文档一并打包，便于老师独立查看。
