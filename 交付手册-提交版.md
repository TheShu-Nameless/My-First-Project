# 门诊预约与智能分诊辅助系统 - 交付手册（提交版）

## 1. 手册目的

本手册用于指导老师/验收人员快速完成系统部署、启动、功能验收与故障排查。  
按照本手册操作，可在 Windows 环境完成项目复现。

## 2. 交付清单

本项目建议提交以下内容：

1. 前后端完整源码目录（`client`、`server`、`database`）
2. 启动脚本（`01`~`05` 及 `start-*.bat`）
3. 项目文档：`项目文档-老师提交版.md`（及导出的 `项目文档-提交版.docx`）
4. 交付手册：`交付手册-提交版.md`（及导出的 `交付手册-提交版.docx`）
5. 产品手册、技术白皮书：`产品手册-提交版.md` / `技术白皮书-提交版.md`（及对应 `.docx`）
6. 静态原型与说明：`门诊预约系统-静态原型.html`、`前端静态HTML说明-提交版.html`（及说明 `.docx`）
7. 数据库脚本：`database/schema.sql`
8. 可选：源码打包 `项目源码-提交版.zip`（由 `node scripts/generate-deliverables.mjs` 生成，已排除 `node_modules`、`.git`、`.tools` 等）

重新生成上述 `.docx` 与 `.zip`：在项目根目录执行 `node scripts/generate-deliverables.mjs`（需先 `cd tools/gen-docx && npm install` 安装依赖）。

## 5. 部署与启动步骤

## 5.1 首次部署（必须先做）

1. 解压源码到本地目录（例如 `Documents/tcm-ai-review-system`）
2. 进入项目根目录，双击：
   - `01-首次运行-初始化数据库.bat`
3. 初始化完成后，双击：
   - `02-日常启动-本机和校内访问.bat`
4. 浏览器访问：
   - `http://localhost:11999`

## 5.2 日常启动

- 本机/校内访问：`02-日常启动-本机和校内访问.bat`
- 校内+校外访问：`03-日常启动-校外和校内都可访问.bat`

> 注意：脚本窗口关闭即服务停止。

## 5.3 AI 配置

- 配置在线 AI Key：`04-设置AI密钥并测试.bat`
- 切换本地免费 AI：`05-一键切换Ollama-免费本地AI.bat`

## 6. 功能验收步骤（建议）

验收顺序如下：

1. 打开登录页并使用管理员登录
2. 查看首页概览与公告
3. 进入挂号预约页，完成一次预约创建
4. 进入 AI 分诊页，输入症状并生成建议
5. 进入后台管理，创建一条公告
6. 退出并重新登录，验证数据仍可读取

判定标准：

- 页面可访问、无白屏
- 登录成功且权限生效
- 预约与公告可创建
- AI 分诊可返回结果（在线模型或本地规则）

## 7. 静态页面验收（作业要求）

静态原型为根目录单文件：`门诊预约系统-静态原型.html`（内嵌样式与脚本，无其它附属文件）。

验收方式：

1. 双击 `门诊预约系统-静态原型.html`
2. 点击「登录（演示跳转）」进入首页
3. 使用顶部链接切换「首页 / 挂号预约 / AI分诊」，「退出」返回登录
4. 浏览器地址栏哈希会随页面变化（如 `#home`、`#appointments`），支持前进后退

> 说明：静态页面仅用于展示原型，不连接后端数据库。

## 8. 安全与稳定性说明

当前版本已实现：

- 登录限流（防暴力尝试）
- 关键操作审计日志（`audit_log`）
- API 请求 `request_id` 跟踪
- 慢接口日志记录
- AI 请求超时控制
- 前端 GET 自动重试
- 密码最小长度校验（至少 8 位）

## 9. 常见问题与处理

- 前端打不开  
  检查 `02` 脚本是否在运行，端口 `11999` 是否被占用。

- 登录失败  
  检查后端是否启动、数据库是否可连、账号密码是否正确。

- AI 分诊无结果  
  先执行 `04` 重新配置 AI key；也可切换 `05` 使用本地 Ollama。

- 校外访问失败  
  使用 `03` 启动并确保 tunnel 窗口未关闭。

- 关闭 Cursor 后访问失败  
  属于正常现象，需重新运行启动脚本恢复服务。

## 10. 维护建议

- 定期备份数据库（`tcm_ai_review`）
- 不要在提交包中包含 `server/.env`（防止密钥泄露）
- 生产环境建议使用独立服务器并配置 HTTPS
