# 门诊预约与智能分诊辅助系统 - 项目文档（提交版）

## 1. 项目简介

本项目是一个面向校医院/实训门诊场景的全栈系统，覆盖预约挂号、医生排班、就诊记录、AI 分诊建议、公告与后台管理等核心流程。  
目标是提升门诊管理效率、减少人工分诊压力，并为学生患者提供更顺畅的就诊体验。

## 2. 技术栈与运行环境

### 2.1 技术栈

- 前端：Vue3 + Vite + Element Plus + Pinia + Vue Router
- 后端：Node.js + Express
- 数据库：MySQL 8.x
- AI：OpenAI 兼容接口（已支持 DeepSeek / DashScope / Ollama）

## 4. 主要功能模块

- 登录与注册
- 首页概览（统计 + 公告）
- 挂号预约（创建/取消/改约）
- 医生排班管理
- 就诊记录管理
- AI 分诊（文本输入 -> 推荐科室与建议）
- 后台管理（用户维护、公告维护）

## 5. 启动与使用流程

## 5.1 首次运行

1. 运行 `01-首次运行-初始化数据库.bat`
2. 运行 `02-日常启动-本机和校内访问.bat`
3. 浏览器访问 `http://localhost:11999`

## 5.2 日常使用

- 本机/校内访问：`02-日常启动-本机和校内访问.bat`
- 校外+校内访问：`03-日常启动-校外和校内都可访问.bat`

> 注意：启动后脚本窗口不能关闭，关闭即服务停止。

## 5.3 AI 配置

- 一键设置 API Key：`04-设置AI密钥并测试.bat`
- 一键切换本地免费 AI（Ollama）：`05-一键切换Ollama-免费本地AI.bat`

## 6. 安全与稳定性说明（当前版本）

本项目已完成以下增强：

- 登录限流（防暴力尝试）
- 关键操作审计日志（`audit_log`）
- 全局请求 `request_id` 追踪
- 慢接口日志输出
- AI 请求超时控制
- 前端 GET 请求自动重试（弱网提升可用性）
- 密码强度下限校验（至少 8 位）

## 7. 角色权限说明

- `admin`：系统最高权限（用户维护、公告维护、业务管理）
- `staff`：值班/运营权限（排班、公告等）
- `doctor`：就诊记录相关权限
- `patient`：预约、查看个人就诊与分诊

## 8. 数据库核心表

- `users`：用户与角色信息
- `department`：科室信息
- `doctor`：医生信息
- `schedule`：排班信息
- `appointment`：预约订单
- `medical_record`：就诊记录
- `inquiry`：AI 问询记录
- `announcement`：公告
- `system_settings`：系统配置
- `audit_log`：操作审计日志

## 9. 常见问题排查

- 访问不了前端：确认前端脚本是否在运行，端口是否被占用（`11999`）
- 登录失败：确认账号密码、后端是否启动、数据库是否正常
- AI 不工作：检查 `04-设置AI密钥并测试.bat` 是否成功；若失败可先切到 `Ollama`
- 校外打不开：确认使用了 `03-日常启动-校外和校内都可访问.bat` 并保持窗口常驻
